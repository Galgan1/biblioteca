BLENDER MANUAL

CAPÍTULO 4: Cycles, Cor e Python (Headless)

Blender Foundation

O motor físico + a API bpy permitem renderizar cenas inteiras por linha de comando, sem abrir a interface.

GPU OptiX na ordem certa

No headless, ativar a GPU exige ordem: compute_device_type='OPTIX' → get_devices() → habilitar cada device. Fora dessa ordem, o Blender silenciosamente cai pra CPU.

OptiX usa os RT/Tensor cores: 40–80% mais rápido que CUDA + denoiser por IA.

AgX: cor de cinema

O AgX (padrão do Blender 4.x) comprime os altos como uma câmera real — sem estourar branco nem desbotar cores saturadas. Nunca entregue em 'Standard'.

Workflow: ajuste a EXPOSIÇÃO, não troque o look, para corrigir brilho.

bpy + --background

blender --background --python script.py renderiza sem GUI. O script cria cena, materiais, luz, câmera e dá render — perfeito para automação por livro.

Produção: renderize para sequência de imagens (PNG/EXR), nunca direto p/ .mp4 (um crash não perde tudo).

Lições-Chave do Capítulo 4

  • Ativar GPU OptiX no bpy exige a ordem correta, senão cai pra CPU.
  • AgX é obrigatório para foto-realismo; ajuste exposição, não o look.
  • bpy + --background automatiza render de cenas inteiras.
